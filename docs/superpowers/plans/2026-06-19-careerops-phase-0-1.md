# CareerOps Phase 0–1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up the CareerOps repository skeleton, a Docker-backed backend stack with health checks, and one real feature (`UserProfile`) taken end-to-end through every layer to prove the toolchain before any domain work.

**Architecture:** Clean Architecture (Domain → Application → Infrastructure → Api), ASP.NET Core Minimal APIs, EF Core code-first against PostgreSQL, Mapster, FluentValidation, RFC 7807 `ProblemDetails`. Frontend is a host-only React/Vite/TypeScript app whose typed API client is generated by orval from the runtime OpenAPI document. Compose runs Postgres + API; the daily inner loop is `dotnet watch` (host) + Vite (host).

**Tech Stack:** .NET 10 / C#, EF Core 10 (Npgsql), FluentValidation, Mapster, Serilog, Microsoft.AspNetCore.OpenApi + Scalar, xUnit + FluentAssertions. React 19 + Vite + TypeScript, TanStack Query, React Hook Form, Zod, Tailwind CSS + shadcn/ui, orval, date-fns. Docker Compose. `just` command runner.

## Global Constraints

These apply to **every task**. Values are copied from the knowledge base + PRD. The choices this plan surfaced (framework, OpenAPI tooling, PK type, profile validation, DDD scope, CLI-first) are **confirmed** and recorded in `docs/knowledge-base/03-decisions.md` as **D14–D19**.

- **Target framework: .NET 10 (LTS), C# latest, `Nullable` enabled, `ImplicitUsings` enabled.** (D14)
- **OpenAPI: built-in `Microsoft.AspNetCore.OpenApi`, document at `/openapi/v1.json`; Scalar UI at `/scalar/v1`. orval reads `/openapi/v1.json`.** Supersedes the old `/swagger/v1/swagger.json` URL (already updated in `05-feedback-loop.md`). (D15)
- **Primary keys: `int` identity (PostgreSQL `GENERATED … AS IDENTITY`), except `UserProfile`, a fixed single row with `Id = 1`.** (D16)
- **`UserProfile` validation (absent from PRD §20): `FullName` required (≤200); `Email` valid email if present; `LinkedInUrl`/`GitHubUrl`/`PortfolioUrl` absolute http(s) URL if present; `TargetSalaryMin ≥ 0` if present; `TargetSalaryCurrency` 3-letter ISO code if present.** (D17)
- **Pragmatic/tactical DDD (D18):** ubiquitous language, aggregates as consistency boundaries, behaviour on entities where it protects an invariant; **no** repositories, MediatR, or domain-event infra until a slice needs them. See `06-engineering-practices.md`.
- **CLI-first (D19):** use the `dotnet` CLI (projects, references, packages, EF migrations, tool manifest) and `npm`/`npx` (vite, shadcn, orval) for all scaffolding/deps/codegen; do not hand-author `.sln`/`.csproj`/`package.json` versions. Source code and CLI-unowned config files are hand-edited.
- API style: **Minimal APIs**, one `MapGroup` module per resource; **every endpoint sets an `operationId` via `.WithName(...)`** (orval depends on it). (D1, `04-conventions.md`)
- Mapping: **Mapster** `IRegister` configs in `Application/Common/Mapping`; no inline mapping in endpoints. (D2)
- Data access: **direct EF Core via `IAppDbContext`; no generic repositories.** (D3)
- Frontend API client: **orval-generated into `frontend/src/lib/api/`; never hand-edited; committed.** Time-boxed at S1.2 — if not clean by end of S1.2, fall back to openapi-typescript + thin fetch (½ day max). (D4)
- Enums persist as **ints with pinned explicit values**; never reorder/renumber. (No enums in these three slices, but the rule stands.) (D5)
- DB: **snake_case** via `EFCore.NamingConventions`; **UTC `timestamptz`**, property/column names end in `AtUtc`; EF code-first migrations committed to git, one per slice; **no soft delete, no JSON columns** in baseline. (`04-conventions.md`)
- Time: **never call `DateTime.UtcNow`**; inject `IClock`. `CreatedAtUtc`/`UpdatedAtUtc` set centrally in `SaveChangesAsync`.
- Errors: **RFC 7807 `ProblemDetails`** everywhere — 400 `ValidationProblemDetails`, 404, 409, 500 (no stack trace, no secrets).
- Runtime: **frontend never in Docker**; everything frontend self-contained under `frontend/`. Compose = Postgres + API only. (D10, D11)
- Connection strings: container API uses `Host=careerops-postgres`; host `dotnet watch` uses `Host=localhost` (via `appsettings.Development.json`). Postgres port `5432` is published to the host.
- Ports: web `5173` (host Vite), api `8080`, postgres `5432`.
- Secrets: **never commit `.env` or keys**; AI keys via env only, never logged. Synthetic data only. (PRD §19.3)
- Git: commit per task; conventional-commit subjects; migrations committed with their slice; branch off `main`.
- Per-slice Definition of Done (the gate for closing S1.1 and S1.2): builds; `just up` runs; orval client regenerated + committed; data persists across restart; bad input → `ProblemDetails`; `just verify` passes; phase-appropriate tests pass; manual usability check.

**Prerequisites on the developer machine:** Docker Desktop, .NET 10 SDK, Node 22 LTS, and the `just` command runner (`winget install Casey.Just` / `scoop install just` / `choco install just`).

Create `backend/src/CareerOps.Api/HealthChecks/DatabaseHealthCheck.cs`:

```csharp
using CareerOps.Application.Common;
using Microsoft.Extensions.Diagnostics.HealthChecks;

namespace CareerOps.Api.HealthChecks;

public sealed class DatabaseHealthCheck(IAppDbContext db) : IHealthCheck
{
    public async Task<HealthCheckResult> CheckHealthAsync(
        HealthCheckContext context, CancellationToken cancellationToken = default)
    {
        var canConnect = await db.CanConnectAsync(cancellationToken);
        return canConnect
            ? HealthCheckResult.Healthy("Database reachable")
            : HealthCheckResult.Unhealthy("Database unreachable");
    }
}
```

using CareerOps.Api.HealthChecks;

using Microsoft.AspNetCore.Diagnostics.HealthChecks;

var builder = WebApplication.CreateBuilder(args);

        => Database.CanConnectAsync(cancellationToken);

        RuleFor(r => r.TargetSalaryMin).GreaterThanOrEqualTo(0).When(r => r.TargetSalaryMin.HasValue);

using CareerOps.Application.Common;
using CareerOps.Domain.UserProfiles;

using Microsoft.EntityFrameworkCore;

namespace CareerOps.Application.Settings;

public sealed class UserProfileService(IAppDbContext db)
{
    private const int SingletonId = 1;

    public async Task<UserProfileDto> GetAsync(CancellationToken ct = default)
    {
        var profile = await db.UserProfiles.FirstOrDefaultAsync(p => p.Id == SingletonId, ct);
        if (profile is null)
        {
            profile = new UserProfile { Id = SingletonId, FullName = "" };
            db.UserProfiles.Add(profile);
            await db.SaveChangesAsync(ct);
        }
        return profile.Adapt<UserProfileDto>();
    }

    public async Task<UserProfileDto> UpdateAsync(UpdateUserProfileRequest request, CancellationToken ct = default)
    {
        var profile = await db.UserProfiles.FirstOrDefaultAsync(p => p.Id == SingletonId, ct);
        if (profile is null)
        {
            profile = new UserProfile { Id = SingletonId };
            db.UserProfiles.Add(profile);
        }
        request.Adapt(profile);     // maps writable fields onto the tracked entity
        profile.Id = SingletonId;   // guard: never let mapping change the key
        await db.SaveChangesAsync(ct);
        return profile.Adapt<UserProfileDto>();
    }
}

public sealed class ValidationFilter<T> : IEndpointFilter
{
    public async ValueTask<object?> InvokeAsync(EndpointFilterInvocationContext ctx, EndpointFilterDelegate next)
    {
        var validator = ctx.HttpContext.RequestServices.GetService<IValidator<T>>();
        var arg = ctx.Arguments.OfType<T>().FirstOrDefault();
        if (validator is not null && arg is not null)

             .AddEndpointFilter<ValidationFilter<UpdateUserProfileRequest>>();

        return group;
    }
}

```ts
const base = import.meta.env.VITE_API_BASE_URL;

export const apiClient = async <T>(config: {
  url: string; method: string; data?: unknown; params?: Record<string, unknown>;
}): Promise<T> => {
  const url = new URL(config.url, base);
  if (config.params) for (const [k, v] of Object.entries(config.params))
    if (v != null) url.searchParams.set(k, String(v));
  const res = await fetch(url, {
    method: config.method,
    headers: config.data ? { "Content-Type": "application/json" } : undefined,
    body: config.data ? JSON.stringify(config.data) : undefined,
  });
  if (!res.ok) throw await res.json().catch(() => ({ status: res.status }));
  return (res.status === 204 ? undefined : await res.json()) as T;
};
```
