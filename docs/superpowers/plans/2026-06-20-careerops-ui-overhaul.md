# CareerOps UI/UX Overhaul Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebuild the CareerOps frontend into a modern Jira/Linear/Trello-style workspace — a shadcn sidebar app shell, a dnd-kit kanban board (+ list) for job leads, slide-over forms, and a dashboard with a pipeline bar — with **zero backend changes**.

**Architecture:** Frontend-only slice ("Phase 2.5") on the installed shadcn/ui (`radix-nova`) system. Adopt the shadcn component set + a Sidebar shell; Job Leads becomes a board where dragging a card issues an **optimistic** `PUT /api/job-leads/{id}` status change (reusing the Phase-2 endpoint); create/edit moves into Sheet/Dialog slide-overs. All data still flows through the orval-generated TanStack Query hooks.

**Tech Stack:** React 19, Vite 8, TypeScript 6, Tailwind v4, shadcn/ui (`radix-nova`, lucide), TanStack Query v5, React Hook Form, react-router v8, **@dnd-kit/core + @dnd-kit/utilities** (new), **sonner** (new). `npm`/`npx` only (D19). No Recharts.

## Global Constraints

- **Frontend-only.** No changes under `backend/`, no API/contract/migration changes. The drag-to-change-status path **reuses** `PUT /api/job-leads/{id}` (D24/D26) — never add a status endpoint.
- **shadcn is the library** (PRD §10.2, installed). Add components with the `shadcn` CLI (`npx shadcn@latest add …`) — they reuse the already-installed `radix-ui` primitives. **Never hand-author** generated component files (D19). `lib/api/` stays orval-generated (D19, `04-conventions.md`).
- **No frontend test runner exists** (tests are backend xUnit). The automated gate per task is **`npm run typecheck` + `npm run build`** (the frontend half of `just verify`); correctness of interactive behaviour (drag, slide-overs) is confirmed by a **manual check** noted in the task. Do **not** add vitest/jest/playwright — unrequested scope; validation stays server-authoritative (D23).
- **Server-authoritative validation (D23)** unchanged: forms submit and display the API's 400 `ProblemDetails`; generated Zod stays unwired.
- **Enums are `number` on the wire (D5);** `lib/enums.ts` owns int→label **and** int→color maps (`04-conventions.md`). `id`/`companyId`/`salaryMin`/`salaryMax`/`fitScore` are typed `number | string | null` by orval — coerce with `Number(...)` when passing path ids; pass numeric body fields through as-is.
- **Deferred (do NOT add):** Recharts, dark-mode toggle, applications board (Phase 3). Tokens for dark mode exist but no toggle.
- **Clean code:** KISS/YAGNI/DRY; small focused files under `features/<area>/` and `components/`; comment the non-obvious *why* only.

- [ ] **Step 2: Create `useLeadMutations.ts`**

```ts
import { useState } from "react";
import { useQueryClient } from "@tanstack/react-query";
import { toast } from "sonner";
import {
  useCreateJobLead, useUpdateJobLead, getGetJobLeadsQueryKey,
} from "@/lib/api/job-leads/job-leads";
import type { CreateJobLeadRequest, JobLeadDto } from "@/lib/api/model";
import { toUpdateRequest } from "./leadRequest";

type CachedLeads = { data: JobLeadDto[] } | undefined;

const readErrors = (e: unknown): string[] => {
  const problem = (e as { data?: { errors?: Record<string, string[]> } }).data;
  return problem?.errors ? Object.values(problem.errors).flat() : ["Save failed."];
};

export function useSaveLead() {
  const qc = useQueryClient();
  const create = useCreateJobLead();
  const update = useUpdateJobLead();
  const [errors, setErrors] = useState<string[]>([]);

  const save = async (req: CreateJobLeadRequest, editingId?: number) => {
    setErrors([]);
    try {
      if (editingId != null) {
        const { companyId, ...rest } = req;
        await update.mutateAsync({ id: editingId, data: { ...rest, companyId: Number(companyId) } });
      } else {
        await create.mutateAsync({ data: req });
      }
      qc.invalidateQueries({ queryKey: getGetJobLeadsQueryKey() });
      toast.success(editingId != null ? "Lead updated" : "Lead added");
    } catch (e) {
      setErrors(readErrors(e));
      throw e;
    }
  };

  return { save, pending: create.isPending || update.isPending, errors };
}

export function useUpdateLeadStatus() {
  const qc = useQueryClient();
  const key = getGetJobLeadsQueryKey();
  const mutation = useUpdateJobLead({
    mutation: {
      onMutate: async ({ id, data }) => {
        await qc.cancelQueries({ queryKey: key });
        const prev = qc.getQueryData(key) as CachedLeads;
        qc.setQueryData(key, (old: CachedLeads) =>
          old ? { ...old, data: old.data.map((l) => (Number(l.id) === id ? { ...l, status: data.status } : l)) } : old,
        );
        return { prev };
      },
      onError: (_e, _vars, ctx) => {
        const prev = (ctx as { prev?: CachedLeads } | undefined)?.prev;
        if (prev) qc.setQueryData(key, prev);
        toast.error("Could not move the lead — reverted.");
      },
      onSettled: () => { qc.invalidateQueries({ queryKey: key }); },
    },
  });

  const changeStatus = (lead: JobLeadDto, status: number) =>
    mutation.mutate({ id: Number(lead.id), data: toUpdateRequest(lead, { status }) });

  return { changeStatus };
}
```
> The `update` path strips `newCompanyName` (not on `UpdateJobLeadRequest`) and coerces `companyId`. `JobLeadForm` only sets `newCompanyName` for new leads, so edits always carry a concrete `companyId`.

export function JobLeadsBoard({ leads, onEdit, showClosed }: Props) {

  const columns = showClosed ? [...ACTIVE_STATUSES, ...CLOSED_STATUSES] : ACTIVE_STATUSES;

  const onDragStart = (e: DragStartEvent) => setDragging((e.active.data.current?.lead as JobLeadDto) ?? null);
  const onDragEnd = (e: DragEndEvent) => {
    setDragging(null);
    const lead = e.active.data.current?.lead as JobLeadDto | undefined;
    const overId = e.over?.id?.toString();
    if (!lead || !overId?.startsWith("col-")) return;
    const target = Number(overId.slice(4));
    if (target !== lead.status) changeStatus(lead, target);
  };

  return (
    <DndContext sensors={sensors} onDragStart={onDragStart} onDragEnd={onDragEnd}>
      <div className="flex gap-4 overflow-x-auto pb-2">
        {columns.map((s) => <BoardColumn key={s} status={s} leads={byStatus(s)} onEdit={onEdit} />)}
      </div>
      <DragOverlay>{dragging ? <LeadCard lead={dragging} onEdit={() => {}} /> : null}</DragOverlay>
    </DndContext>
  );
}

  const [open, setOpen] = useState(false);

  const [errors, setErrors] = useState<string[]>([]);

  const companies = response?.data ?? [];
  const invalidate = () => qc.invalidateQueries({ queryKey: getGetCompaniesQueryKey() });
  const readErrors = (e: unknown): string[] => {
    const p = (e as { data?: { errors?: Record<string, string[]> } }).data;

  };

  const onSubmit = async (req: CreateCompanyRequest) => {
    setErrors([]);

      if (editing) await update.mutateAsync({ id: Number(editing.id), data: req });
      else await create.mutateAsync({ data: req });
      setOpen(false); invalidate(); toast.success(editing ? "Company updated" : "Company added");
    } catch (e) { setErrors(readErrors(e)); }
  };

  const onDelete = async (c: CompanyDto) => {
    if (!confirm(`Delete ${c.name}? This also deletes its job leads.`)) return;
    await remove.mutateAsync({ id: Number(c.id) });
    invalidate(); toast.success("Company deleted");
  };

  if (isLoading) return <Skeleton className="h-64 w-full" />;

  return (
    <div className="space-y-6">
      <div className="flex items-center justify-between">
        <h1 className="text-2xl font-semibold">Companies</h1>
